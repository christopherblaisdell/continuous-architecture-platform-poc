## Summary

Add `elevation_gain_meters` and `elevation_loss_meters` fields to the GET /trails/{trail_id} response in svc-trail-management.

## Description

Currently the trail response from svc-trail-management only includes `distance_km` and `estimated_duration_hours`. Ranger staff and guest-facing trip planning tools need elevation data to properly assess trail difficulty and provide gear recommendations.

Elevation data (gain and loss in meters) is already captured and stored in the `trails` database table but is not mapped to the Trail DTO returned by the API. This ticket requests exposing these two existing database fields in the API response.

**Current Response Fields (partial):**
- `trail_id`
- `name`
- `distance_km`
- `estimated_duration_hours`
- `difficulty_rating`

**Requested Additional Fields:**
- `elevation_gain_meters` (nullable decimal)
- `elevation_loss_meters` (nullable decimal)

## Acceptance Criteria

1. GET /trails/{trail_id} response includes `elevation_gain_meters` as a nullable numeric field
2. GET /trails/{trail_id} response includes `elevation_loss_meters` as a nullable numeric field
3. GET /trails (list endpoint) also returns the new fields for each trail in the collection
4. Swagger/OpenAPI specification is updated to document the new fields
