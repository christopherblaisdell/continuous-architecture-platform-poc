## Summary

Enable unregistered guests to use self-service check-in kiosks via reservation lookup and identity verification

## Description

### Problem Statement

Currently, only guests who have registered NovaTrek accounts can use the self-service check-in kiosks at adventure base camps. Unregistered guests -- including those who booked through travel partners (ExploreMore, TrailFinder, WildPass), gift card recipients, and companions added to existing reservations -- must queue for staff-assisted check-in at the service desk.

This creates several problems:

1. **Long wait times**: During peak arrival windows (typically 7:00-9:00 AM), unregistered guests wait an average of 22 minutes for staff-assisted check-in. Registered guests using kiosks complete check-in in under 3 minutes.
2. **Staffing pressure**: Base camps must maintain 2-3 additional check-in staff during peak hours specifically to handle unregistered guest volume, representing approximately $840K annually across all locations.
3. **Guest friction**: 34% of unregistered guests report dissatisfaction with the check-in experience in post-adventure surveys. This is the number one complaint category for partner-booked guests.
4. **Conversion loss**: Only 12% of unregistered guests who check in via staff assistance convert to registered NovaTrek accounts. Industry benchmarks suggest self-service flows with account creation prompts achieve 35-45% conversion.
5. **Safety compliance bottleneck**: Staff must manually verify waiver completion and gear assignments during check-in, adding 4-6 minutes per guest compared to the automated kiosk flow.

### Proposed Solution

Add a reservation lookup capability to the self-service check-in kiosk flow. Unregistered guests can identify themselves using four verification fields:

- Last name (as on reservation)
- Reservation confirmation code
- Adventure date
- Number of participants

Upon successful verification, the system grants temporary kiosk access for the duration of the check-in process, allowing the guest to complete waiver signing, gear assignment confirmation, and adventure briefing acknowledgment through the standard kiosk workflow.

### Business Impact

- Estimated 60% reduction in staff-assisted check-in volume
- Projected annual savings of $500K in staffing costs
- Expected improvement in guest satisfaction scores for partner-booked guests
- Increased account conversion opportunity through post-check-in registration prompt

## Acceptance Criteria

1. Unregistered guests can initiate self-service check-in by selecting "Check in with Confirmation Code" on the kiosk welcome screen
2. The system verifies guest identity using all four fields (last name, confirmation code, adventure date, participant count) -- all must match
3. Upon successful verification, the guest receives a temporary session (30-minute expiry) to complete the kiosk check-in flow
4. If the reservation was booked through a partner and direct lookup fails, the system falls back to partner integration verification
5. Guests with incomplete safety waivers are prompted to complete digital waiver signing as part of the kiosk flow
6. Gear assignments linked to the reservation are displayed for guest confirmation
7. Failed verification attempts are rate-limited (5 attempts per kiosk per 15-minute window) with clear user messaging
8. A temporary guest profile is created (or matched to existing) to support the kiosk session without requiring full account registration
9. All verification attempts (successful and failed) are logged for security audit
10. Check-in analytics differentiate between registered-guest kiosk, unregistered-guest kiosk, and staff-assisted check-ins

### Comment 3 - Marcus Chen (Security Reviewer) - 2026-01-24

Security concerns flagged:
1. The verification fields must be submitted via POST body, never as URL query parameters -- PII exposure risk
2. We need rate limiting to prevent reservation enumeration attacks
3. Temporary kiosk sessions must be time-bounded and scoped to a single kiosk device
4. Failed attempt logging is mandatory for SOC monitoring

I will need to review the final endpoint specification before approving.
